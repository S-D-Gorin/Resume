## Bot Sprotect

### Sprotect Bots

Антиспам-платформа для Telegram-групп

Разработал антиспам-бота для Telegram-групп, предназначенного для автоматической модерации, удаления нежелательного контента и защиты чатов от спама, рекламы и подозрительной активности.

Проект построен как контейнерная микросервисная платформа с разделением bot-layer и backend-логики.

---

# Основные возможности

### Автоматическая модерация Telegram-групп

* удаление спама
* удаление рекламных сообщений
* антифлуд-логика
* проверка подозрительных пользователей
* реакция на триггеры и паттерны сообщений
* защита групп без участия администратора

### Управление через админ-панель

* настройка правил модерации
* управление whitelist / trusted users
* управление шаблонами сообщений
* контроль работы системы
* просмотр данных и событий

### Автоматические фоновые процессы

* периодические рассылки
* delayed deletion сообщений
* сервисные задачи по расписанию
* очистка служебных данных

# Контейнерная схема

## 1. Bot Gateway Layer

Отдельный контейнер с PyTelegramBotAPI:

* принимает события Telegram через infinity_polling()
* сериализует входящие сообщения
* отправляет события в backend по API (JSON)

### Результат:

изоляция Telegram transport-слоя от бизнес-логики.

---

## 2. Backend Application Layer

Контейнер с Django + Gunicorn:

* обработка входящих сообщений
* антиспам-логика
* модерация чатов
* работа с данными админки
* отправка ответов пользователям
* удаление сообщений
* управление Telegram API

Дополнительно:
используются внешние Python / FastAPI сервисы для расширенной обработки сообщений.

### Особенность архитектуры:

в backend используется тот же BOT_TOKEN, что позволяет выполнять управляющие действия напрямую:

* deleteMessage
* sendMessage
* moderation actions

---

## 3. Data Layer

PostgreSQL:

* данные групп
* настройки модерации
* пользователи
* события
* служебные данные

---

## 4. Reverse Proxy Layer

NGINX:

* входная точка системы
* reverse proxy
* SSL termination
* маршрутизация запросов

---

## 5. Scheduler Layer

Cron контейнер:

* периодические задачи
* рассылки
* удаление старых сообщений
* вызовы backend endpoints
* обслуживание системы
